Welcome To GitHub

Git is a code hosting platform for version control and collabrotion. Lets learn GitHub essentials like repositories, branches, commits and pull requests. Git pull request is a popular way to reivew code.

Introduction:

Step 1. Create a Repository

Repositories are used to organize our projects. We can add foldres, files, data sets, images.

To Create a New Repository

Click '+' button at the right corner and select 'New Repository'. Name and Add description to the repository. And finally Initialize the repository with a 'README'.

Step 2. Create a Branch

Branching is the way to work on diferent version of a repository at one time. We use branches to make edits before commiting to the 'master' branch which is the main branch.

To Create a New Branch

Go to the new repository you have just created, click the drop down of Branch: master and create a new branch and click on 'create branch'.

Step 3. Make and commit changes

Saved changes are called Commits. Commit messages capture the history of your changes

To make a commit changes

Click on README.md file and click on pencil icon at the right corner, edit the file and a commit message that describes the changes and click on commit changes button.

Step 4. Open a Pull request

As soon as you make a commit, you can open pull request and start a discussion, even before the coding is done

To make Pull request

click on pull request button, select the branch to compare with master. Click on the create pull request.

Step 5. To Merge the Pull Request

This is the time to bring all your changes together

Click merge pull request button to merge the changes to master branch. Click Confirm merge.Go ahead and delete the branch, since its changes have been incorporated, with the Delete branch button in the purple box.
